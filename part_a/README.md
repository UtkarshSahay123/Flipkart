# Part A - AI Prediction & Recommendation Engine

This folder contains the Part A workflow from the project brief: dataset preparation, feature engineering, model training, prediction, and the FastAPI service used by the other teams.

## Structure

- `data/raw/` - source dataset files
- `data/processed/` - cleaned or feature-engineered data
- `notebooks/` - EDA, feature engineering, and training notebooks
- `src/` - reusable Python modules for prep, features, training, prediction, and rules
- `models/` - trained model artifacts generated by `src/train.py`
- `api/` - FastAPI application
- `tests/` - API and helper tests

## Setup

```bash
pip install -r requirements.txt
```

## Train

```bash
python -m part_a.src.train --data-path part_a/data/raw/traffic_events.csv
```

You can swap in a different CSV later by changing `--data-path` without editing the training code.

## Run API

```bash
<folder path>/.trafficenv/Scripts/python.exe -m uvicorn part_a.api.main:app --reload --port 8001
```

The website command center now points to this service at `http://localhost:8001`.

## Endpoint

`POST /predict`

Input fields:
- `event_type`
- `location`
- `road_issue`
- `hour_of_day`
- `day_of_week`
- `is_weekend`
- `expected_crowd`

Retraining artifacts:
- `part_a/data/processed/traffic_events_processed.csv`
- `part_a/models/model.pkl`
- `part_a/models/metrics.json`

Response fields:
- `congestion_level`
- `delay_minutes`
- `affected_radius_km`
- `risk_score`
- `resources`
